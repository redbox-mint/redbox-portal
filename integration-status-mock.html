IntegrationStatusComponent — UI Reference Implementation

Markup and classes match the ReDBox portal: Bootstrap 5, Font Awesome 4, and form component conventions. This component renders one row per integration with status, timestamps, and optional key results (e.g., minted DOI).

State 1: In progress (polling active)

Shows when an integration job is queued or running. The widget displays a spinner in the header and badge.

Integrations Checking status…
  • In progress
    DOI (DataCite) Started 11 Jun 2026, 10:42

State 2: Success with key result

Shown after an integration completes successfully. Displays duration and a key result (e.g., the minted DOI as a link).

Integrations
  • Success
    DOI (DataCite) Completed 11 Jun 2026, 10:42 (in 2.3s)
    Draft DOI minted: 10.25903/abc-123-456

State 3: Failed (with sanitized message)

Shows when an integration fails. The message is human-readable and sanitized—no raw error details or HTTP status codes (those remain admin-only in the audit log). The record save itself succeeded.

Integrations
  • Failed
    DOI (DataCite) Failed 11 Jun 2026, 10:42
    The DataCite service could not be reached. Your record has been saved; contact an administrator if this persists.

State 4: Multiple integrations, mixed states

When multiple integrations are configured (e.g. DOI + Figshare), they each appear as a separate row. Shows the real-world case where one completes while another is in progress.

Integrations
  • Success
    DOI (DataCite) Completed 11 Jun 2026, 10:42
  • In progress
    Figshare Started 11 Jun 2026, 10:43

State 5: No integration activity yet (empty state)

When a record is created or no integrations have run yet. The component may also hide entirely in create mode (before record is saved and has an OID).

Integrations

No integration activity recorded for this record yet.

Behavior notes
  • On form load: Component fetches integration status once (if record has an OID).
  • After save success: Refreshes status after ~1.5s delay (audits are queued; gives agenda time to write the initial 'started' entry).
  • Polling: While any integration is 'started', polls every pollIntervalMs (default 5000ms) up to maxPollAttempts times (default 60 attempts). Stops when all are terminal or limit reached.
  • Access control: Only users with edit access to the record see the widget. Admin-only detail (errorDetail, requestSummary, responseSummary) remains on the admin audit tab.
  • In create mode: Component hides itself (no OID yet). Becomes visible after first save.
  • ARIA live region: Set to polite for updates, assertive for errors, so screen readers announce status changes.

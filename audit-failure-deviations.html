Integration Failure Notifications — Implementation Deviations

1. Cache TTL for failure state

Plan: Set the cache TTL to windowSeconds or longer to keep recovery state.

Implementation: TTL is set exactly to throttle.windowSeconds (default 300). If a success occurs after the throttle window expires, the failure state will have been evicted and no recovery alert will be sent. This is the simplest interpretation; users can extend the window to keep state longer if recovery alerts are needed.

2. Slack/Webhook channel stubs

Plan: Define Slack and Webhook adapter stubs/types but mark deferred.

Implementation: No stub classes were created. The channel registry (getChannel) only maps 'email'; unknown types are logged and skipped. Adding Slack/Webhook channels requires writing a class implementing IntegrationNotificationChannel and registering it in the Map, which matches the "pluggable" intent.
